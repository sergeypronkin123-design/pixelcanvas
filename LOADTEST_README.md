# Нагрузочный тест PixelStake

## 🚀 Быстрый старт (5 минут)

### 1. Установите Locust

Откройте терминал (в папке проекта) и выполните:

```bash
pip install locust websockets
```

Если `pip` не работает, попробуйте:
```bash
python -m pip install locust websockets
```

### 2. Запустите тест

В терминале (в корне проекта):

```bash
locust -f loadtest_simple.py --host=https://pixelcanvas-api.onrender.com
```

### 3. Откройте веб-интерфейс

В браузере откройте: **http://localhost:8089**

### 4. Настройте параметры

На странице Locust укажите:
- **Number of users**: начните с **50**
- **Ramp up**: **5** users per second
- **Host**: оставьте `https://pixelcanvas-api.onrender.com`

Нажмите **Start swarm**.

## 📊 Что смотреть

Locust покажет в реальном времени:

- **RPS** (requests per second) — сколько запросов обрабатывает сервер
- **Response times** — время ответа (медиана, P95, P99)
- **Failures** — ошибки (должно быть 0%)

### На что обращать внимание:

✅ **Здоровый сервер:**
- Median response time < 500ms
- P95 < 2000ms
- Failures = 0%

⚠️ **Начинает напрягаться:**
- P95 > 3000ms
- Появляются 500 ошибки

🔴 **Не справляется:**
- Failures > 5%
- Timeouts (таймауты)
- Response time > 10 секунд

## 🎯 План тестирования

Постепенно увеличивайте нагрузку:

| Этап | Users | Ramp up | Время | Что проверяем |
|---|---|---|---|---|
| 1 | 50 | 5/sec | 2 мин | Работает ли вообще |
| 2 | 100 | 10/sec | 3 мин | Базовая нагрузка |
| 3 | 250 | 10/sec | 3 мин | Средняя |
| 4 | 500 | 10/sec | 5 мин | Пик для Free tier |
| 5 | 1000 | 20/sec | 5 мин | Уже сложно |

**Важно**: между тестами делайте паузу 1-2 минуты — Render Free засыпает
после неактивности и первые запросы будут медленными.

## 📝 Сохранить результаты

Можно запустить с сохранением в файл:

```bash
locust -f loadtest_simple.py --host=https://pixelcanvas-api.onrender.com \
    --headless -u 500 -r 10 -t 5m --html=report.html
```

Это запустит без веб-интерфейса: 500 users, ramp 10/sec, 5 минут, сохранит HTML отчёт.

## ⚠️ Важные предупреждения

1. **НЕ ТЕСТИРУЙТЕ СИЛЬНО на production** — может исчерпать лимиты Neon Free (100 connections)

2. **Тест создаёт реальные записи в БД** — будут пользователи с email `loadtest_*`. 
   После теста можно удалить через SQL:
   ```sql
   DELETE FROM users WHERE email LIKE 'loadtest_%';
   DELETE FROM pixels WHERE user_id IS NULL;
   ```

3. **Render Free засыпает** после 15 минут неактивности. Первый запрос "разбудит"
   сервер — это займёт 30-60 секунд.

4. **Первый тест покажет плохие результаты** потому что сервер холодный. 
   Делайте второй тест через 2-3 минуты — вот его результаты реальны.
